# **CARLA_ws 진행 현황 및 방향성 정리**

## **1) 워크스페이스 개요**
- ~/CARLA_ws는 ROS2 기반 자율주행 파이프라인을 CARLA 시뮬레이터로 확장하기 위한 통합 워크스페이스임.
- 현재는 로컬 bag 기반 검증 단계에서, 시뮬레이터 실시간 데이터 기반 검증 단계로 넘어가는 전환 구간

## **2) 패키지별 현재 상태**
**src/my_pkg**
- 로컬 ros bag 데이터 기반으로 TF 체인 구조를 개편
- SLAM으로 맵 생성까지 수행
- Nav2는 매우 기초 수준만 확인:
    - Nav2 서버/스택 구동 확인
    - /cmd_vel 실제 발행 및 주행 제어는 미확인
**src/my_pkg_carla_bridge**
- 기존 별도 워크스페이스에서 수행했던 내용을 필요한 부분만 추려 이식한 패키지
- 포함된 핵심 기능:
    - ROS bridge 테스트
    - CARLA 시뮬레이터 연동
    - 고정 루트 주행 시뮬레이션 구성

## **3) 앞으로의 진행 방향**
**1. 이식 검증 단계**
    my_pkg_carla_bridge가 현재 워크스페이스에서 정상 동작하는지 우선 확인
    - CARLA 실행
    - scripts/run_fixed_route.sh 실행
    - 기본 토픽/노드 동작 점검
**2. 실시간 센서 기반 전환 단계**
    1번 검증 완료 후, 기존 bag 기반으로 하던 SLAM/Nav2 테스트를
    CARLA 실시간 LiDAR 데이터 기반으로 전환하여 재검증

**3. 최종 목표 단계**
    시뮬레이터 상에서 ROS2 통신으로 센서 데이터를 수집하고,
    SLAM으로 맵 생성 후, 생성된 맵 기반으로 차량이 자율주행하도록 통합 완성

## **4) 현재 이슈 (추후 해결 필수)**
**1. ros2 launch 비정상 종료 (exit code -11 추정)**
    - 세그폴트 계열 가능성 높음
    - 패키지/의존성/빌드 아티팩트 불일치 가능성 우선 의심
    - 우선순위 높음 (기본 실행 안정성 이슈)
**2. bag 기반 Nav2 연동 품질 저하**
    - 동작이 불안정하고 삐걱거리는 현상 존재
    - 원인 미확정 (시간 동기/TF/센서 주기/파라미터 등 점검 필요)
    - 실시간성 확보 전에는 Nav2 고도화가 제한적
**3. SLAM 맵 기반 경로 계획 품질 문제**
    - LiDAR 포인트 사이 빈 공간으로도 경로를 잡는 현상 확인
    - 비용맵/장애물 레이어/맵 해상도/인플레이션 설정 점검 필요
    - 추후 실시간 시뮬레이터 데이터 기준으로 재평가 예정

# **5) 요약**
현재는 bag 기반 검증을 끝내고 CARLA 실시간 검증으로 넘어가는 단계이며,
단기적으로는 bridge 이식 안정화, 중기적으로는 실시간 LiDAR 기반 SLAM/Nav2 검증,
최종적으로는 ROS2 + SLAM + Nav2 통합 자율주행이 목표다.
